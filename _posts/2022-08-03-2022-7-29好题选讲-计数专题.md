## T0 树上最远点对
&emsp; ~~热身题qwq~~

&emsp; 传送门：[树上最远点对](https://www.51nod.com/Challenge/Problem.html#problemId=1766)

&emsp; $tag$：**树上最远点对的结论？？？**

### 分析

&emsp; 对于一个点集，其中的最远点对是 $(x_1, y_1)$，另一个点集，最远点对是 $(x_2, x_2)$，那么这两个点集合并后最远点对必定在 $(x_1, y_1)$ ，$(x_1, y_2)$，$(x_2, y_1)$，$(x_2, y_2)$，$(x_1, x_2)$，$(y_1, y_2)$ 中产生。

&emsp; 这个结论很好证明，这里就不证了，具体的可以去看看关于树的直径两边 $dfs$ 的做法的正确性证明。

&emsp; 所以我们就可以用线段树，从一个点开始，向上合并。复杂度就是 $O(n\log^2 n)$ 的。


### 代码

```cpp
// 这个代码会 T，但是不是算法的问题，是我一开始写的倍增 lca 所以每次是 log 的
// 改成 ST 表求 lca 每次 O(1) 就能过，但是我懒的改qwq
#include<bits/stdc++.h>
using namespace std;
#define int long long
#define in read()
#define MAXN 100100
#define MAXM MAXN << 1
#define ls(p) (p << 1)
#define rs(p) (p << 1 | 1)

inline int read(){
	int x = 0; char c = getchar();
	while(c < '0' or c > '9') c = getchar();
	while('0' <= c and c <= '9'){
		x = x * 10 + c - '0'; c = getchar();
	}
	return x;
}

int n = 0; int m = 0;
int tot = 0;
int first[MAXN] = { 0 };
int   nxt[MAXM] = { 0 };
int    to[MAXM] = { 0 };
int value[MAXM] = { 0 };
inline void add(int x, int y, int weight){
	nxt[++tot] = first[x];
	first[x] = tot; to[tot] = y;
	value[tot] = weight;
}

int ddd[MAXN] = { 0 };
int dep[MAXN] = { 0 };
int f[MAXN][50] = { 0 };
void prework(int x, int fa){
	dep[x] = dep[fa] + 1;
	for(int i = 0; i <= 20; i++) f[x][i + 1] = f[f[x][i]][i];
	for(int e = first[x]; e; e = nxt[e]){
		int y = to[e];
		if(y == fa) continue;
		f[y][0] = x; ddd[y] = ddd[x] + value[e];
		prework(y, x);
	}
}

int Lca(int x, int y){
	if(dep[x] < dep[y]) swap(x, y);
	for(int i = 20; i >= 0; i--){
		if(dep[f[x][i]] >= dep[y]) x = f[x][i];
		if(x == y) return x;
	}
	for(int i = 20; i >= 0; i--)
		if(f[x][i] != f[y][i])
			x = f[x][i], y = f[y][i];
	return f[x][0];
}
int dis(int x, int y) { return ddd[x] + ddd[y] - 2 * ddd[Lca(x, y)]; }

struct Tnode{
	int x, y;
	int l, r;
}t[MAXN << 2];

inline void up(int p){
	int x1 = t[ls(p)].x, y1 = t[ls(p)].y;
	int x2 = t[rs(p)].x, y2 = t[rs(p)].y;
//	printf("  ll = %d lr = %d rl = %d rr = %d\n", t[ls(p)].l, t[ls(p)].r, t[rs(p)].l, t[rs(p)].r);
//	printf("  x1 = %d y1 = %d x2 = %d y2 = %d\n", x1, y1, x2, y2);
	int d = 0;
	if(dis(x1, y1) > d) t[p].x = x1, t[p].y = y1, d = dis(x1, y1);
	if(dis(x1, y2) > d) t[p].x = x1, t[p].y = y2, d = dis(x1, y2);
	if(dis(x1, x2) > d) t[p].x = x1, t[p].y = x2, d = dis(x1, x2);
	if(dis(x2, y1) > d) t[p].x = x2, t[p].y = y1, d = dis(x2, y1);
	if(dis(x2, y2) > d) t[p].x = x2, t[p].y = y2, d = dis(x2, y2);
	if(dis(y1, y2) > d) t[p].x = y1, t[p].y = y2, d = dis(y1, y2);
//	printf("  x = %d y = %d\n", t[p].x, t[p].y);
}

void build(int p, int l, int r){
	t[p].l = l, t[p].r = r;
	if(l == r) { t[p].x = t[p].y = l; return; }
	int mid = (l + r) >> 1;
	build(ls(p), l, mid); build(rs(p), mid + 1, r);
//	printf("l = %d r = %d\n", l, r);
	up(p);
}

pair<int, int> mx(pair<int, int> a, pair<int, int> b) {
	return dis(a.first, a.second) > dis(b.first, b.second) ? a : b; }
pair<int, int> query(int p, int l, int r){
	if(t[p].l == l and t[p].r == r) return make_pair(t[p].x, t[p].y);
	int mid = (t[p].l + t[p].r) >> 1;
	if(r <= mid) return query(ls(p), l, r);
	else if(l > mid) return query(rs(p), l, r);
	else{
		pair<int, int> a = query(ls(p), l, mid);
		pair<int, int> b = query(rs(p), mid + 1, r);
		int x = 0, y = 0, d = 0;
		int x1 = a.first, y1 = a.second;
		int x2 = b.first, y2 = b.second;
		if(dis(x1, y1) > d) x = x1, y = y1, d = dis(x1, y1);
		if(dis(x1, y2) > d) x = x1, y = y2, d = dis(x1, y2);
		if(dis(x1, x2) > d) x = x1, y = x2, d = dis(x1, x2);
		if(dis(x2, y1) > d) x = x2, y = y1, d = dis(x2, y1);
		if(dis(x2, y2) > d) x = x2, y = y2, d = dis(x2, y2);
		if(dis(y1, y2) > d) x = y1, y = y2, d = dis(y1, y2);
		return make_pair(x, y);
	}
}

signed main(){
	n = in;
	for(int i = 1; i < n; i++){
		int x = in, y = in, w = in;;
		add(x, y, w), add(y, x, w);
	} prework(1, 0);
	m = in; build(1, 1, n);
	for(int i = 1; i <= m; i++){
		int a = in, b = in;
		int c = in, d = in;
		pair<int, int> node1 = query(1, a, b);
		pair<int, int> node2 = query(1, c, d);
		int x1 = node1.first, y1 = node1.second;
		int x2 = node2.first, y2 = node2.second;
		int dis1 = dis(x1, x2), dis2 = dis(x1, y2);
		int dis3 = dis(y1, x2), dis4 = dis(y1, y2);
		int dd = max(dis1, max(dis2, max(dis3, dis4)));
		cout << dd << '\n';
	}
	return 0;
}
```

## T1 字符串变换

&emsp; 因为我没找到哪个 $OJ$ 上有这道题，所以就放个题目大意在这里啦：

&emsp; $tag$：**钦定在 ~~xxxx~~ 的时候统计答案？？？**

### 题目大意

&emsp; 往长度为 $n$ 的字符串的任意位置插入 $m$ 个任意小写字母，问最终得到的不同字符串个数 $n, m \leq 1e6$。

### 分析

&emsp; 我们发现，这道题的难点其实在于去重，就比如原串为 $abaa$，$m = 4$，时有很多种插入方法让最终的字符串变成 $abaaabaa$。所以我们要考虑一种统计答案的方法，使得在统计的过程中不会出现重复。

&emsp; 所以我们钦定原串在最后时计算贡献。

&emsp; 什么意思呢，就是对于 $abaaabaa$ 这个生成的串，我们把原串标成红色，只有在是这个样子的时候我们才统计答案：

$$ abaa \color{red} abaa $$

&emsp; $\color{red} ab \color{a} aa \color{red} a \color{a} ba \color{red} a$，或者 $\color{red}abaa \color{a} abaa$ 这种原串不在最后的是不统计答案的。

&emsp; 那么我们现在考虑如何计数，我们枚举原串的第一个字母的位置 $i$，那么对于 $1 \sim i-1$ 就可以任意放字母了，所以我们肯定有意向这个东西：

$$ \sum_{i = 1}^{m}26^{i - 1} $$

&emsp; 那么对于原串后面的字符来说，后面还有 $n + m - i$ 个空位我们可以任意放置剩下的 $n - 1$ 个字符，所以式子中还肯定要乘上一个组合数：

$$ \sum_{i = 1}^m 26^{i - 1} \begin{pmatrix} n + m - i \\ n - 1 \end{pmatrix} $$

&emsp; 然后就是对于原串放置的第一个字符和第二个字符之间的位置来说，它是不能放置第一个字符的，因为如果放的时第一个字符，生成的串中就有一个串比原串更靠后了。

&emsp; 同理，第二个字符到第三个字符之间不能填第二个字符，第三个到第四个之间不能填第三个字符，以此类推下去。我们发现，这样的话我们剩下位置的选择其实都是 $25$ 个，因为约束条件都只是不能填某一个。所以式子还要乘上一个 $25^{m - i - 1}$

$$ ans = \sum_{i = 1}^m 26^{i - 1} \begin{pmatrix} n + m - i \\ n - 1 \end{pmatrix}25^{m - i - 1} $$

&emsp; 这个题就做完了。

## T2 Help Yourself G

&emsp; 传送门：[[USACO20FEB]Help Yourself G](https://www.luogu.com.cn/problem/P6146)

&emsp; $tag$：**跟上一道题一样？？？**

### 分析

&emsp; 这个题的难点和上一道题一样，都是怎么去重，所以我们沿用上一题的思路钦定在某一情况下统计答案。

&emsp; 我们根据题目意思可以发现， 每种情况下的一个连通块对答案都有 $1$ 的贡献，但是我们不能保证这个连通块只被计算一次贡献，因为在考虑每一个小的区间的时候我们可能将拼成同一个联通块的小区间的个数当作这个连通块的贡献，这样就算重了。

&emsp; 所以我们**钦定只有一个连通块最右端的小区间（也就是决定了连通块的右端点的区间）我们才计算贡献。**

&emsp; 假设某一个小区间 $[l, r]$ 是一个连通块最右边的区间，那么对于任意其他连通块 $[l', r']$，应该要满足：

$$ l' > r \lor r' < r $$

&emsp; 那么我们假设满足 $r' < r$ 的小区间有 $x$ 个，满足 $l' > r$ 的小区间有 $y$ 个，那么这一个小区间产生的贡献就是 $2^{x + y}$（因为这 $x + y$ 个满足条件的区间都是可选可不选的，而剩下的都是一定不能选的）。

&emsp; 那么我们只需要预处理一个 $s_i$ 表示 $r < i$ 的个数，然后枚举，快速幂就搞定了。

### 代码

```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long
#define in read()
#define MAXN 200200
#define MOD 1000000007

inline int read(){
	int x = 0; char c = getchar();
	while(c < '0' or c > '9') c = getchar();
	while('0' <= c and c <= '9'){
		x = x * 10 + c - '0'; c = getchar();
	}
	return x;
}

int n = 0;
struct Trange{
	int l, r;
}seg[MAXN];
int s[MAXN] = { 0 };
bool comp(Trange a, Trange b) { return a.l < b.l; }

int qp(int x, int y){
	int ans = 1;
	for(; y; y >>= 1){
		if(y & 1) ans = (1ll * ans * x) % MOD;
		x = (1ll * x * x) % MOD;
	}
	return ans;
}

signed main(){
	n = in; int ans = 0;
	for(int i = 1; i <= n; i++) seg[i].l = in, seg[i].r = in;
	for(int i = 1; i <= n; i++) s[seg[i].r]++;
	sort(seg + 1, seg + n + 1, comp);
	for(int i = 1; i <= 2 * n; i++) s[i] = (s[i] + s[i - 1]) % MOD;
	for(int i = 1; i <= n; i++){
		ans = (2 * ans + qp(2, s[seg[i].l - 1])) % MOD;
	} cout << ans << '\n';
	return 0;
}
```

## T3 

&emsp; $tag$ ：**成对统计？？？**

&emsp; 还是没找到哪个 $OJ$ 上有这个题，所以：

### 题目大意

&emsp; 给定 $n, m$，要求统计长度为 $2m$ 的序列个数，要求序列中元素为 $n$ 的因数，且 $\prod\limits_{i = 1}^{2m}a_i \leq n^m$， $n < 1e9，m < 100$。

### 分析

&emsp; 我们发现这个约束条件有点难搞，那么我们还是可以考虑考虑不满足这个条件的方案数，也就是满足：

$$ \prod_{i = 1}^{2m}a_i > n^m $$

&emsp; 的个数。

&emsp; 我们经过一些观察，我们把弄出来的 $a_i$ 都变成 $\frac n {a_i}$，那么就变成了：

$$ \prod_{i = 1}^{2m}\frac n {a_i}  = \frac{n^{2m}}{\prod\limits_{i = 1}^{2m}a_i} $$

&emsp; 又因为 $\prod\limits_{i = 1}^{2m}a_i > n^m$，所以：

$$ \prod_{i = 1}^{2m}\frac n {a_i}  = \frac{n^{2m}}{\prod\limits_{i = 1}^{2m}a_i} < n^m $$

&emsp; 所以我们发现，对于一个 $\prod a_i > n^m$ 的方案和一个 $\prod a_i < n^m$ 的方案是一一对应的，也就是说这两种情况的方案数是相等的。

&emsp; 于是我们就能发现，答案就等于：

$$ \frac{总方案数 + 等于 n^m 的方案数}2 $$

&emsp; 总方案数很好处理，就是 $(2m)^{\sigma_0(n)}$，其中 $\sigma_0(n)$ 表示 $n$ 的约数个数。然后就是要处理等于 $n^m$ 的方案数，我们考虑对 $n$ 进行质因数分解，那么：

$$ n^m = \prod_{i}p_i^{mk_i} $$

&emsp; 我们考虑每一个质因数怎么分配，那么这就是一个方程的解的问题了，就是对于每一个质因数 $p_i$，我们考虑怎样分把 $p_i^{mk_i}$ 拆成几个数的乘积的方案数，也就是 $2m$ 个 $x_j \geq 0$，使得 $\prod_j p_i^{x_j} = p_i^{mk_i}$，也就是：

$$ \sum_j x_j = mk_i $$

&emsp; 这个就是简单的插板问题了，然后对于每一个 $p_i$ 都做一次，最后再乘起来就好了。

## T4 卡农

&emsp; 传送门：[[HNOI2011] 卡农](https://www.luogu.com.cn/problem/P3214)

&emsp; $tag$：**递推 + 容斥？？？**

### 分析

&emsp; 这里有几个限制条件：

1. 无空集
2. 任意两个集合不相等
3. 所有元素出现偶数次（所有集合加起来统计）

&emsp; 首先我们把每一个集合都看成一个数字，然后最后得到的就是一个序列，序列的方案数除以 $m!$ 显然就是集合的方案数，所以我们首先考虑序列。

&emsp; 我们考虑一个条件一个条件处理。首先只考虑无空集，令 $f_i$ 表示 $m = i$ 时的答案，且只考虑第一个条件时：

$$ f_i = A_{2^n - 1}^i $$

&emsp; 也就是从 $2^n - 1$ 种子集中选 $i$ 个 ~~这个甚至不用递推~~。

&emsp; 然后加上第二个和第三个条件，我们尝试从前面转移到 $f_i$，但是我们经过考虑会发现一个矛盾的地方：前 $f_{i - 1}$ 也是一个完整的方案，也就是说里面的元素个数为偶数。如果我要保证 $f_i$ 里面元素个数也是偶数的话，那就只能加空集了。这就和不能加入空集矛盾了。

&emsp; 所以这里我们不能直接从 $f_{i - 1}$ 转移。

&emsp; 所以我们要转移的话前面就一定不能合法。也就是说我们可以用刚才没有满足二、三条件的方程 "生成出" 前 $i - 1$ 个数，那么对于每一个方案来说，都有一些数的个数是奇数，也有一些数的个数是偶数，而且在这一种情况下， 第 $i$ 位的生成方式也就确定下来了，也就是把奇数个数的数放到第 $i$ 个集合里面就好了，这样就能保证出现次数是偶数了。

&emsp; 于是 $f_i$ 就等于 $A_{2^n - 1}^{i - 1}$....... 吗？

&emsp; 显然不是，因为这样构造还有一些问题，如果说我们前面 $i - 1$ 个集合的数的个数已经都是偶数了，那么第 $i$ 个集合就只能是空集，这就与前面我们的约束条件矛盾了。还有可能第 $i$ 个集合在前面已经出现过了，那么也就矛盾了。

&emsp; 于是我们考虑容斥掉这些不合法的答案。

&emsp; 我们令：

1. $i$ 为空集有 $x$ 种
2. 与 $i$ 重复有 $y$ 种

&emsp; 所以总的答案就是：

$$ f_i = A_{2^n - 1}^{i - 1} - x - y $$

&emsp; 那么我们考虑怎么求出 $x$ 和 $y$。

&emsp; 显然 $x$ 就是 $f_{i - 1}$，原因我们上面分析的时候也说过了（也就是解释不能直接从 $f_{i - 1}$ 转移的那一段）。

&emsp; 然后 $y$ 就有一点复杂了。我们假设 $i$ 与前面的 $j$ 重复了，并且我们知道 $f_i$ 这里是合法的，所以如果我们把 $i$ 和 $j$ 的集合删掉，剩下的 $i - 2$ 个集合就是合法的了（偶数减偶数等于偶数）。那么剩下的东西的方案数就可以用 $f_{i - 2}$ 表示了，那么：

$$ y = (i - 1)[2^{n} - 1 - (i - 2)]f_{i - 2} $$

&emsp; 前面的两个系数 $(i - 1)$ 表示枚举 $j$ 的位置，然后 $[2^n - 1 - (i - 2)]$ 表示枚举集合 $i$（枚举它具体长什么样）。

&emsp; 于是就有：

$$ f_i = A_{2^n - 1}^{i - 1} - f_{i - 1} - (2^n - i + 1)(i - 1)f_{i - 2} $$

&emsp; 然后就直接递推就好了。


### 代码
```cpp
#include<bits/stdc++.h>
using namespace std;
#define MAXN 1001000
#define p (int(1e8 + 7))

int f[MAXN] = { 0 };
int n = 0; int m = 0;
int x = 1; int mx = 1;                                // x = 2^n, mx = m!
int A[MAXN] = { 0 };

int qp(int a, int b){
	int ans = 1;
	for(; b; b >>= 1){
		if(b & 1) ans = 1ll * ans * a % p;
		a = 1ll * a * a % p;
	}
	return ans;
} 

int main(){
	cin >> n >> m;
	for(int i = 1; i <= n; i++) x = x * 2 % p;
	x = (x - 1 + p) % p; A[0] = 1;
	for(int i = 1; i <= m; i++)
		A[i] = 1ll * A[i - 1] * ((x - i + 1 + p) % p) % p,
		mx = 1ll * mx * i % p;
	f[0] = 1, f[1] = 0;
	for(int i = 2; i <= m; i++)
		f[i] = (A[i - 1] - f[i - 1] + p - 1ll * (i - 1) % p * (x - i + 2 + p) % p * f[i - 2] % p + p) % p;
	cout << 1ll * f[m] * qp(mx, p - 2) % p << '\n';
	return 0;
}
```
