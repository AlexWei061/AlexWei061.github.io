# Day 1

## T1 格雷码

### 题目大意

&emsp;  按照格雷码的规则生成 $n$ 为二进制的排列，并且输出编号为 $k$ 的这个排列。

&emsp; $n$ 位格雷码不止一种，下面给出其中一种格雷码的生成算法：

1. 1 位格雷码由两个 1 位二进制串组成，顺序为：0，1。
2. $n + 1$ 位格雷码的前 $2^n$ 个二进制串，可以由依此算法生成的 $n$ 位格雷码（总共 $2^n$ 个 $n$ 位二进制串）按**顺序**排列，再在每个串前加一个前缀 0 构成。
3. $n + 1$ 位格雷码的后 $2^n$ 个二进制串，可以由依此算法生成的 $n$ 位格雷码（总共 $2^n$ 个 $n$ 位二进制串）按**逆序**排列，再在每个串前加一个前缀 1 构成。

&emsp; 按该算法，2 位格雷码可以这样推出：

1. 已知 1 位格雷码为 0，1。
2. 前两个格雷码为 00，01。后两个格雷码为 11，10。合并得到 00，01，11，10，编号依次为 0 ~ 3。

&emsp; 同理，3 位格雷码可以这样推出：

1. 已知 2 位格雷码为：00，01，11，10。
2. 前四个格雷码为：000，001，011，010。后四个格雷码为：110，111，101，100。合并得到：000，001，011，010，110，111，101，100，编号依次为 0 ~ 7。


### 分析

&emsp; 我们根据规则就可以发现，$n$ 为的二进制格雷码的编号是从 $0 \sim 2^n - 1$，并且对于第编号 $i$ 和编号 $2^n - i$ 的两个二进制串来说**除了最高位相反外，其余位置都相同**。

&emsp; 然后我们又可以发现。如果把下标写成 $n$ 位的二进制数的话，就像这样：

1. 0(0)， 1(1)
2. 00(00)， 01(01)， 11(10)， 10(11)
3. 000(000)，001(001)，011(010)，010(011)，110(100)，111(101)，101(110)，100(111)
$\vdots$

&emsp; 我们就能发现编号满足 "对于第编号 $i$ 和编号 $i + 2^{n - 1}$ 的两个二进制串来说**除了最高位相反外，其余位置都相同**" 的规律，所以我们考虑找一找编号和答案的关系。

&emsp; 再观察观察我们可以发现，因为这个产生排列的规则是前半部分有 $2^n$ 个，后半部分也有 $2^n$ 个，所以如果编号 $k$ 的某一位是 $1$，也就说明这个答案从前面构造过来的时候经历了一次反转。所以我们考虑这样：

&emsp; 对于一个 $k$，把它写成 $n$ 位二进制串，每遇到一个 "$1$"，我们就把后面的所有串反转一遍，也就是 $0$ 变成 $1$，$1$ 变成 $0$，也就是异或上一个全 $1$ 串。这样做就相当于从现有的串开始依次往回回溯到最初始串。这就是一个 $O(\log n)$ 的做法了，显然是能过掉这个题的，但是我们还是可以继续优化这个算法。

&emsp; 我们考虑不在遇到一个 $1$ 就反转一遍，二十给每个数打上一个反转标记。

&emsp; 我们考虑考虑这个反转标记有没有什么规律，很明显的，情况分为四种：

1. 当前数为 $0$ 且 $rev\_tag = 0$，那么说明当前这一位数没有被反转，那它后面的也就不用被反转了，所以它下一位的 $rev_tag$ 也就等于 $0$。
2. 当前数为 $0$ 且 $rev\_tag = 1$，那么说明当前这个数被反转了变为 $1$，同理，他后面的也就不用反转（因为要翻两次就相当于不翻了），所以它下一位数的 $rev\_tag = 0$。
3. 当前数为 $1$ 且 $rev\_tag = 0$，那么说明当前这意味没有被反转，所以他后面的要被 $1$ 反转一次，也就是它下一位的 $rev\_tag = 1$.
4. 当前数为 $1$ 且 $rev\_tag = 1$，那说明当前数要被反转成 $0$，那么说明后面的数还是要被反转一次，所以他下一位的 $rev\_tag$ 还是 $1$。

&emsp; 把所有情况写出来就可以发现，$rev\_tag$ 的值就和上一位的 $k$ 的值是一样的，而且 $k$ 直接异或上 $rev_tag$ 就是答案，所以答案就是 $k \oplus (k >> 1)$。于是求出答案就变成 $O(1)$ 的了 ~~虽然输出仍然是 log 的~~。

### 代码

```cpp
#include<bits/stdc++.h>
using namespace std;
#define int unsigned long long

int n = 0; int k = 0;

int main(){
	cin >> n >> k;
	k ^= k >> 1;
	while(~--n) cout << (k >> n & 1);
	return 0;
}
```

## T2 括号树

### 题目大意

&emsp; 题目太长了，懒得概括了，放个传送门吧：[CSP-S2019 Day1 T2](https://www.luogu.com.cn/problem/P5658)

### 分析

&emsp; 显然从根向下 $dfs$，我们每到新的一个点就会加进来一个字符，现在就有两种情况：

1. 加进来的字符是左括号，显然这不会对答案有任何影响，就直接不管。
2. 加进来的字符是右括号，我们考虑这个括号对答案的贡献。首先这个右括号所匹配到的左括号肯定是唯一的，而且这就形成了一坨合法的串（从最右端到左边的某个位置）。我们假设加进来的右括号匹配到的左括号的下标为 $i$，那么要想和这一坨合法串组成新的合法串的话左边的合法串的结尾就一定是 $i - 1$，这样就能拼到一起形成一个新的大串。那么新加进来的这个右括号所产生的答案就是 $1 + num_{i - 1}$，其中 $num_{i - 1}$ 是以 $i - 1$ 结尾的串中合法串的个数。也就是说设新家进来的右括号的位置是 $j$，那么 $num_j = num_{i - 1} + 1$。

&emsp; 这样一来这道题就变得十分简单了。

### 代码
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long
#define in read()
#define MAXN 500500
#define MAXM MAXN << 2

inline int read(){
	int x = 0; char c = getchar();
	while(c < '0' or c > '9') c = getchar();
	while('0' <= c and c <= '9'){
		x = x * 10 + c - '0'; c = getchar();
	}
	return x;
}

int n = 0; int tot = 0;
int first[MAXN] = { 0 };
int   nxt[MAXM] = { 0 };
int    to[MAXM] = { 0 };
int value[MAXN] = { 0 };
inline void add(int x, int y){
	nxt[++tot] = first[x];
	first[x] = tot; to[tot] = y;
}

int top = 0, temp = 0, ans = 0;
int stk[MAXN] = { 0 };
int num[MAXN] = { 0 };
int lst[MAXN] = { 0 };                                                // 上一个能匹配到的右括号
void dfs(int x){
	stk[++top] = value[x];
	if(value[x] == 0){
		int p = top - 1;
		if(lst[p] != -1) p = lst[p] - 1;                              // 如果 p 不是左括号
		if(p != 0 and stk[p] == 1){                                   // 存在而且是左括号(能匹配上)
			num[top] = 1;
			if(p != 1 and lst[p - 1] != -1)                           // 不是第一个且前一个不是左括号
				lst[top] = lst[p - 1],
				num[top] += num[p - 1];
			else lst[top] = p;                                        // 否则就是第一个
			temp += num[top];
		}
		else lst[top] = -1, num[top] = 0;                             // 是左括号
	}
	else lst[top] = -1, num[top] = 0;
	ans = ans ^ (x * temp);
	for(int e = first[x]; e; e = nxt[e])
		dfs(to[e]);
	temp -= num[top], top--;
}

signed main(){
	n = in; char c;
	for(int i = 1; i <= n; i++){
		c = getchar();
		while(c != '(' and c != ')') c = getchar();
		if(c == '(') value[i] = 1;
		else value[i] = 0;
	}
	for(int i = 1; i < n; i++){
		int y = in;
		add(y, i + 1);
	} dfs(1);
	cout << ans << '\n';
	return 0;
}
```


## T3 树上的数

&emsp; 首黑祭 qwq。

### 题目大意

&emsp; 给一颗有 $n$ 个节点的树，每个节点上有两个值，一个编号，一个数字。并且编号和数字都是属于 $[1, n]$ 的且每个数字只都出现一次。

&emsp; 现在有 $n - 1$ 个操作，每次删一条树上的边，同时，这条边连接着的两个点要交换数字。进行完所有操作之后，把所有节点取出来，然后按照数字排序，形成一个节点序列。然后把所有点的编号取出来形成一个数组。

&emsp; 现在要求通过改变操作顺序求出字典序最小的数组，并输出。

### 分析

&emsp; 因为要让字典序最小，所以我们考虑贪心。首先让数字 $1$ 到达尽量小的编号上，再让数字 $2$ 到达满足数字 $1$ 在最小的前提下的最小编号上，以此类推。

&emsp; 我们现在考虑删边的顺序会给答案产生什么影响。我们考虑一个点 $x$，它连向其他很多其他的点 $y_i$，令由 $y_i$ 们组成的集合为 $Y$。我们现在如果先删一条边 $e = (x, y_i), y_i \in Y$，那么中间的点 $x$ 就会和 $y_i$ 交换，此时，其他的点 $y_j \in Y$ 就不可能再得到 $x$ 这个点了。

&emsp; 还有一种特殊情况，我们考虑最后一个删掉边 $e = (x, y_i), y_i \in Y$，那么最终 $x$，原本的位置会变成 $y_i$，其他点也不可能在 $x$ 原来的位置上了。

&emsp; 另外的情况自然就是既不是第一个也不是最后一个删掉的的边了。

&emsp; 于是我们考虑把边分成三种：

1. 第一个删的边称为送出边，效果是送出 $x$。
2. 最后一个删的边为接受边，效果是接受 $y_i$。
3. 中间删掉的边成为转运便，效果是转运 $a$。

&emsp; 其中在转运的过程中可能对删边的顺序有着一定的要求，如果是我们要将向连成链的三个点 $x, y, z$ 中的一个端点 $x$ 送向另一个端点 $z$ 。那么就肯定是先删 $e_1 = (x, y)$，然后立刻删 $e_2 = (y, z)$。我们把这种删了 $e_1$ 立马要删 $e_2$ 的约束条件用符号 $e_1 \to e_2$ 表示。

&emsp; 我们要考虑贪心（尝试把 $x$ 移动到 $y$），我们就要把路径拆分成三个过程：

1. 把 $x$ 送出去
2. 中转
3. $y$ 接收 $x$

&emsp; 我们现在来考虑这三个部分的要求。

&emsp; 首先是把 $x$ 送出去，在前面的描述中我们已经知道，要想送出 $x$ 就一定要使 $Path(x \to y)$ 上的与 $x$ 连接的出边 $e_1$ 是第一个被删掉的（也就是让它变成送出边）。同理，$Path(x \to y)$ 中与 $y$ 相连的边 $e_2$ 必须是接受边，要最后一个删，中转途中还有一些 $e_i \to e_j$ 的条件。

&emsp; 于是我们考虑维护一条偏序链（维护一个点的删边顺序）来维护这些约束条件。

&emsp; 那么我们只需要知道从 $x \to y$ 能不能成功就要满足满足以下的条件不成立。

1. 送出：
a. 第一个位置已经被占用。
b. $e_1$ 已经无法放入第一个位置。
c. 放完 $e_1$ 之后形成了有头有尾的偏序链，且还有边没放进去。
2. 接收：
a. 最后一个位置已经被占用。
b. $e_2$ 已经无法放入最后一个位置。
c. 放完 $e_2$ 之后形成了有头有尾的偏序链，且还有边没放进去。
3. 中转：
a.  条件 $e_i \to e_j$ 与此前的偏序链出现矛盾。
c. 放完 $e_i$ 之后形成了有头有尾的偏序链，且还有边没有放进去。

&emsp; 这样一来，我们从小到大枚举数字 $x$，从小到大枚举编号 $i$，判断能否将 $x$ 的节点送到 $i$ 的节点。这样一来根据我们上面的方法判断，那么判断的复杂度就是 $O(n)$ 的了，于是总的复杂度就是 $O(n^3)$。

&emsp; 但是我们可以优化这个做法，因为加入两次判断能否送达的点 $node_1$ 和 $node_2$ 满足 $dep[lca(node_1, node_2)] < dep[x]$，那么就说明两次判断的路径中有重复。而那一段重复显然只用判断一次就好了。

&emsp; 所以我们考虑，每次我们从数字 $x$ 开始，直接 $dfs$ 一遍，在 $dfs$ 的时候就顺便确定每个点能否成为接收点（包括这两点之间能否成为中转点）。然后找到最小的能到达的就好了。这样对于每一个数字我们就只用 $O(n)$ 了，总的复杂度就是 $O(n^2)$了。

### 代码

```cpp
#include<bits/stdc++.h>
using namespace std;
#define in read()
#define MAXN 2020
#define MAXM MAXN << 2

inline int read(){
	int x = 0; char c = getchar();
	while(c < '0' or c > '9') c = getchar();
	while('0' <= c and c <= '9'){
		x = x * 10 + c - '0'; c = getchar();
	}
	return x;
}

int T = 0;
int n = 0; int tot = 0;
int first[MAXN] = { 0 };
int   nxt[MAXM] = { 0 };
int    to[MAXM] = { 0 };
inline void add(int x, int y){
	nxt[++tot] = first[x];
	first[x] = tot; to[tot] = y;
}

int p[MAXN] = { 0 };                                         // p[idx] = val
int d[MAXN] = { 0 };                                         // i 上还有多少条边没走过
int d1[MAXN] = { 0 };                                        // i 上还有多少条边只被一个数字走进来一次
int d2[MAXN] = { 0 };                                        // i 上还有多少条边只被一个数字走出来一次
int fa[MAXN] = { 0 };                                        // i 的父亲
int from[MAXN] = { 0 };                                      // i 上的点是从哪儿运来的
int ttoo[MAXN] = { 0 };                                      // i 上的点被搬去了哪儿
int st[MAXN][MAXN] = { 0 };                                  // 对于节点 i 来说 (i, j) 为当前偏序链开头的节点
int en[MAXN][MAXN] = { 0 };                                  // 对于节点 i 来说 (i, j) 为当前偏序链结尾的节点
int edge[MAXN][MAXN] = { 0 };                                // -1 : (i, j) 没有被经过过; i : (i, j) 只有一个数字从 i 到 j 经过了一次; j : 只有一个数字从 j 到 i 经过一次; 0 : 其他情况
int flag[MAXN] = { 0 };                                      // 是否可行

inline void init(){
	tot = 0;
	for(int i = 1; i <= n; i++)
		d[i] = d1[i] = d2[i] = 0,
		first[i] = from[i] = ttoo[i] = 0;
}

void dfs(int x, int root){
	for(int e = first[x]; e; e = nxt[e]){
		int y = to[e];
		if(y == fa[x]) continue;
		fa[y] = x, flag[y] = 1;
		if(x != root){
			if(edge[fa[x]][x] == fa[x] or edge[x][y] == x) flag[y] = 0;
			if(edge[fa[x]][x] == 0 or edge[x][y] == 0) flag[y] = 0;
			if(en[x][y] == from[x] and st[x][fa[x]] == ttoo[x] and d2[x] + d1[x] + d[x] * 2 - 2 > 0) flag[y] = 0;
			if(en[x][y] == fa[x]) flag[y] = 0;
		}
		else{
			if(edge[x][y] == x) flag[y] = 0;
			if(edge[x][y] == 0) flag[y] = 0;
			if(flag[x])
				if(en[x][y] == from[x] and d[x] + d1[x] + d2[x] - 1 != 0) flag[y] = 0;
		}
		flag[y] &= flag[x];
		dfs(y, root);
	}
	if(x == root) flag[x] = 0;
	else{
		if(from[x]) flag[x] = 0;
		if(ttoo[x])
			if(en[x][ttoo[x]] == fa[x] and d[x] + d1[x] + d2[x] - 1 != 0) flag[x] = 0;
	}
}

int main(){
	freopen("a.in", "r", stdin);
	T = in;
	while(T--){
		n = in; init();
		for(int i = 1; i <= n; i++) p[i] = in;
		for(int i = 1; i < n; i++){
			int x = in, y = in;
			add(x, y), add(y, x);
			d[x]++, d[y]++;
			edge[x][y] = edge[y][x] = -1;
			st[x][y] = en[x][y] = y;
			st[y][x] = en[y][x] = x;
		}
		for(int i = 1; i <= n; i++){
			for(int j = 1; j <= n; j++) fa[j] = 0;
			flag[p[i]] = 1;
			dfs(p[i], p[i]);
			int res = 0;
			for(int j = 1; j <= n; j++)
				if(flag[j]) { res = j; break; }
			cout << res << ' ';
			from[res] = fa[res];
			while(fa[res] != p[i]){
				if(edge[fa[res]][res] == -1){
					edge[fa[res]][res] = edge[res][fa[res]] = fa[res];
					d[res]--, d2[res]++;
					d[fa[res]]--, d1[fa[res]]++;
				}
				else{
					edge[fa[res]][res] = edge[res][fa[res]] = 0;
					d1[res]--, d2[fa[res]]--;
				}
				int t = res;
				res = fa[res];
				st[res][en[res][t]] = st[res][fa[res]];
				en[res][st[res][fa[res]]] = en[res][t];
			}
			if(edge[fa[res]][res] == -1){
				edge[fa[res]][res] = edge[res][fa[res]] = fa[res];
				d[res]--, d2[res]++;
				d[p[i]]--, d1[p[i]]++;
			}
			else{
				edge[fa[res]][res] = edge[res][fa[res]] = 0;
				d1[res]--, d2[p[i]]--;
			}
			ttoo[p[i]] = res;
		}
		puts("");
	}
	return 0;
}

```
