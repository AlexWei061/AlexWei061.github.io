## T1

### 50pts

&emsp; 一眼 $dp$，转移非常的显然，因为有三种操作所以令 $f_i$ 表示数 $i$ 的最小操作次数，那么：

$$
\begin{aligned} &f_{i +1} = \min\{ f_{i +1}, f_i +1 \} \\ &f_{2i} =  \min \{ f_{2i}, f_i +1 \} \\ & f_{3i} = \min \{ f_{3i}, f_i + 1 \} \end{aligned}
$$

&emsp; 然后这样就可以 $O(n)$ 预处理 钱$1e6$ 的答案然后 $O(1)$ 查询。

### 100pts

&emsp; $n \leq 1e9$，所以我们还需要另外处理 $1e6 \sim 1e9$ 的值，根据这个数据范围就大致能确定处理的复杂度是 $O(\log n)$ 或者 $O(\sqrt n)$，又因为题目中的操作是每次除以一个数，所以大概率复杂度就应该是 $O(\log n)$ 的。

&emsp; 我们考虑搜索，最暴力地搜就是这样：

```cpp
int solve(int x){
	int ans = INFI;
	if(!(x % 2)) ans = min(ans, solve(x / 2) + 1);
	if(!(x % 3)) ans = min(ans, solve(x / 3) + 1);
	ans = min(ans, solve(x - 1) + 1);
	return ans;
}
```

&emsp; 然后我们考虑剪枝，因为刚才我们直接线性预处理的 $1e6$ 的数据，所以我们可以直接剪掉 $1e6$ 一下的搜索：

```cpp
int solve(int x){
	if(x < 1e6 + 100) return f[x];
	int ans = INFI;
	if(!(x % 2)) ans = min(ans, solve(x / 2) + 1);
	if(!(x % 3)) ans = min(ans, solve(x / 3) + 1);
	ans = min(ans, solve(x - 1) + 1);
	return ans;
}
```

&emsp; 再然后我们发现这个爆搜的瓶颈就在于最后一句的 $solve(x - 1) +1$，如果一直执行这个就会使得复杂度爆炸，所以如果能优化掉这个地方我们这个搜索就变成 $\log n$ 的了。

&emsp; 这个时候我们发现，我们最多减两次 $1$ 就会使得当前数变成 $3$ 的倍数，又因为除以 $2, 3$ 是一定比减 $1$ 要优的，所以我们限制最多减两个 $1$，就变成这样：

```cpp
int solve(int x, int del){
	if(x < 1e6 + 100) return f[x];
	int ans = 114;
	if(!(x % 2)) ans = min(ans, solve(x / 2, 0) + 1);
	if(!(x % 3)) ans = min(ans, solve(x / 3, 0) + 1);
	if(del < 2)  ans = min(ans, solve(x - 1, del + 1) + 1);
	return ans;
}
```

&emsp; 然后就可以过掉了。


## T2

### 40pts

&emsp; 简简单单 $O(n^2)$ 预处理出来所有边的长度，然后 $O(n^2\log n^2)$ 跑一边 $kruskal$，就完事了。

### 100pts

&emsp; 因为这个图非常稠密，所以我们考虑在稠密图上更优的 $prim$ 算法，这样就把所有点划分成两个部分，一个是已经在树中的点，标记为 $x \in T$，还有没有在树中的点标记为 $x \in S$。

&emsp; 如果我们每次暴力找 $S$ 和 $T$ 之间的最短边，时间复杂度就是 $O(n^2)$ 的，所以我们需要优化这个暴力做法。

&emsp; 我们考虑边的长度 $val = P&#124;i - j&#124; +a_i + a_j$，这个绝对值就很离谱，所以我们考虑将绝对值拆开，分两种情况：第一种就是 $a_i - Pi$ 第二种是 $a_i + Pi$，对于 $a_i - Pi$ 只有在另一个点 $j > i$ 时才有贡献没另一部分同理，后面我们把这两个东西成为 $A, B$ 两个部分。然后对于两个点 $i < j$ 我们在 $A$ 中取出 $i$ 的值，然后在 $B$ 中取出 $j$ 的值然后加在一起就得到了 $i \to j$ 的距离。

&emsp; 于是我们可以维护 $4$ 颗线段树分别记录 $TA, TB, SA$ 和 $SB$ 的信息，然后再维护第 $5$ 颗线段树，维护 $S$ 与 $T$ 之间的最小边权。

&emsp; 现在我们考虑如果做第 $5$ 颗线段树的 $pushup$ 操作，这里我们要找的是 $S$ 到 $T$ 之间的最小边权，所以一定是从 $SA$ 到 $TB$ 或者从 $TA$ 到 $SB$ 的一条边。所以我们现在对于一个节点 $p$ 分三种情况分类讨论：

1. 答案的两个点的编号都来属于 $(l, mid)$，也就是答案都来源于左儿子。
2. 答案的两个点的编号都来属于 $(mid +1, r)$，也就是答案都来源于右儿子。
3. 答案的两个点的编号一个属于 $(l, mid)$ 另一个属于 $(mid +1, r)$，也就是答案的来源跨过左右儿子。

&emsp; 显然对于前两种情况，我们往下递归就能变成第三种情况，所以很好处理，然后我们来考虑第三种情况怎么做。

&emsp; 我们经过观察发现左儿子里面的点的编号一定小于右儿子里面的点的编号 ~~正常人都能一眼看出来吧qwq~~，所以左儿子里面一定就是查 $A$，右儿子一定就查 $B$，所以只需要分两种情况讨论了：

1. 左儿子里的答案来源于 $T$，那么右儿子里的答案一定来源于 $S$。此时的答案就是 $TA.ls(p) + SB.rs(p)$
2. 左儿子里的答案来源于 $S$，那么右儿子里的答案一定来源于 $T$。此时的答案就是 $SA.ls(p) + TB.rs(p)$

&emsp; 找到这两个答案之后再 $checkmin$ 一下 $pushup$ 操作就做完了。

## T3

&emsp; 又是一道考场上暴力都不会打的题...

&emsp; 转化一下模型，现在有一颗严格 $k$ 叉树，并且有 $n + m$ 个叶子节点，其中 $n$ 个是 $0$，$m$ 个是 $1$。对于每一个非叶子节点来说，她的权值就是她的所有儿子权值的平均值，问有多少种不同的根节点的权值。

&emsp; 假设 $n$ 个 $0$ 的深度分别问 $x_i$，$m$ 个 $1$ 的深度为 $y_i$，那么根节点的权值可以写作：

$$ \sum_{i = 1}^m \frac 1{k^{y_i}} $$

&emsp; 我们发现如果这个这棵树的叶子节点全都是 $1$ 的话，显然这棵树 的根节点的值也就是 $1$，也就是如果一棵树是合法的，那么她就要满足一下的式子：

$$ \sum_{i = 1}^n \frac 1{k^{x_i}} + \sum_{i = 1}^m \frac 1{k^{y_i}} = 1 $$

&emsp; 那么我们考虑证明从这个式子能否说明树就是合法的，显然只有这个式子是不能说明是合法的，因为这是一颗完全 $k$ 叉树，所以要满足 $(n + m - 1) \equiv 0 \pmod {k - 1}$，但是因为题目说了一定满足这个条件所以就不用判断了。

&emsp; 于是乎这个题我们要求的就可以转化成：存在多少个 $z$ 使得：

1. $z$ 可以写成 $n$ 个 $k^{-x_i}$ 的和
2. $1 - z$ 可以写成 $m$ 个 $k^{-y_i}$ 的和

&emsp; 然后观察 $\sum\limits_{i = 1}^{n}k^{-x_i}$ 这个式子，她显然是一个分式，并且这个分式的每一项的分母都是 $k^{x_i}$，所以如果我们把这个数转化成一个 $k$ 进制的数字她就会变得非常漂亮。**因为每一个 $k^{-x_i}$ 就相当于在小数点后第 $x_i$ 为加上一个 $1$。**

&emsp; 那么我们把 $z$ 在 $k$ 进制下的数写成这样 $z = 0.z_1z_2z_3\cdots z_l$，那么 $z$ 能写成 $n$ 个 $k^{-x_i}$ 的和当且仅当满足以下两个条件：

1. $\sum\limits_{i = 1}^l z_i \leq n$
2. $\sum\limits_{i = 1}^l z_i \equiv n \pmod {k - 1}$

&emsp; 第一个式子表示最多加 $k$ 个 $1$，而第二个式子表示进位，**因为我们知道每次发生进位的时候 $\sum z_i$ 就会减少 $k - 1$**，所以 $\sum z_i$ 与 $n$ 模 $k - 1$ 同余。

&emsp; 那么综上所述，我们就是要找出满足以下要求的式子：

1. $0 \leq z_i \leq k - 1$
2. $z_l \geq 0$
3. $\sum\limits_{i = 1}^l z_i \leq n$ 且 $\sum\limits_{i = 1}^l z_i \equiv n \pmod {k - 1}$
4. $\sum\limits_{i = 1}^l (k - 1 - z_i) \leq m - 1$ 且 $\sum\limits_{i = 1}^l (k - 1 - z_i) \equiv m - 1 \pmod {k - 1}$

&emsp; 我们注意到上面的限制就只和 $l$ 和 $\sum z_i$ 有关，所以我们记录 $f_{l, sum}$ 表示当前长度为 $l$，$\sum z_i = sum$ 的不同小数数量，然后转移就好了。
